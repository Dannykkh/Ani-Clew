## [10:37:50] Assistant

Claude Cowork는 **Claude Desktop에서 로컬 파일을 직접 관리하는 에이전트**입니다.

```
Claude Code   = 터미널에서 코딩
Claude Cowork = 데스크톱에서 일반 업무 (파일 정리, 엑셀, 문서 작성 등)
```

**AniClew로 비슷한 게 되냐?** 사실 이미 핵심은 됩니다:

| Cowork 기능 | AniClew | 상태 |
|------------|---------|------|
| 로컬 파일 읽기/쓰기 | Read, Write, Edit | ✅ |
| 폴더 정리/관리 | Bash, LS, Glob | ✅ |
| 웹 검색 | WebSearch, WebFetch | ✅ |
| 병렬 태스크 | KAIROS 데몬 | ✅ 기본 |
| 브라우저에서 사용 | 웹 UI | ✅ |
| **컴퓨터 화면 제어** | ❌ | 없음 |
| **11개 전문 플러그인** | ❌ | 없음 |
| **엑셀/문서 직접 편집** | Notebook만 | 부분 |

**차이점**: Cowork는 **화면을 보고 마우스를 클릭**하는 Computer Use가 핵심이고, AniClew는 **파일/터미널 기반**입니다.

Computer Use를 추가하려면 Playwright나 Chrome DevTools를 연결하면 되는데, 이건 MCP로 이미 가능합니다:

```
.mcp.json에 playwright 또는 chrome-devtools MCP 서버를 등록하면
→ AniClew가 MCP로 연결
→ 브라우저 제어 가능 (클릭, 스크린샷, 입력 등)
```

즉 **MCP 서버만 연결하면 Cowork 같은 기능도 확장** 가능합니다. 추가할까요?

`#tags: claude-cowork, computer-use, 비교, mcp-확장, 가능성`
